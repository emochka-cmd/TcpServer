Проект представляет собой событийно-ориентированный TCP-сервер с применением многопоточности и механизма epoll для эффективной обработки множества клиентских подключений.

Общая структура - cистема построена по модульному принципу, где каждый модуль инкапсулирует свою зону ответственности и не имеет прямых зависимостей от других модулей. Взаимодействие между компонентами осуществляется через чётко определённые интерфейсы. 

Tcp server:  
&emsp;1. Acceptor - отвечает за запуск и инициализацию сервера, привязку к порту, а также принятия новых клиентов. Передает новых клиентов в ClientManager  
&emsp;2. Epoll - реализация событийно ориентированной модели сстемного вызова epoll. Отслеживает активность зарегистрированных файловых дескрипторов и уведомляет сервер о готовности данных к чтению, записи или возникновении ошибок.   
&emsp;3. ClientManager - хранит реестр активных клиентов и управляет их жизненным циклом. Отвечает за добавление, удаление и отслеживание состояния каждого подключения.  
&emsp;4. СommandManager - принимает и интерпретирует команды, поступающие от клиентов, и организует их выполнение. При необходимости делегирует задачи модулю Thread для параллельной обработки.

Thread - модуль выполнения команд. При возможности разбиения задачи она добавляется сюда.  

Client - Представляет абстракцию клиентского подключения. Инкапсулирует файловый дескриптор сокета, буферы чтения/записи и текущее состояние клиента в рамках сессии.  

Json - Модуль загрузки и разбора конфигурационного файла в формате JSON. Предоставляет серверу параметры запуска.
